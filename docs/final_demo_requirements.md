# 期末 Demo 規定

## 目的

應用期中報告的技術，開發一個實際應用。

---

## 限制

- 要能實際 run
- 要應用到期中報告的技術
- 至少 3 個節點
  - 可以同一台機器，三個 Processes；且它們只**透過網路溝通**

---

## 要求

### 功能性

- 要具有意義的應用（不能是 hello world）

### 非功能性（重要）

- 實現至少一項分散式系統模式：容錯、系統偵測、自動回復、隨需伸縮…
- **必須用實驗證明有達成，否則不採計！**
- 未實現本項者，期末展示的滿分只有 80 分
- 必須是自己額外寫的 code，不能直接用 tool
  - 可以改寫他人源碼，但要搞懂來源和改寫程度
  - 請先與老師討論實作的具體想法

參考：https://martinfowler.com/articles/patterns-of-distributed-systems/

---

## 如何繳交（評分依據）

> **未含影片與原始碼 Github 網址者不予計分！**

1. **口頭報告與 demo**
   - 投影片上傳到 Moodle（pdf 格式）
   - Demo 以影片錄製，上傳到雲端，連結附在投影片上

2. **原始碼**
   - 放到小組其中一員的 GitHub 公開 repository
   - 在 `README.md` 中詳述安裝與操作步驟
   - GitHub 網址附在投影片中
